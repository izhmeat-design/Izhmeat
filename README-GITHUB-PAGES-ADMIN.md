# GitHub Pages версия с админкой через ручной GitHub token

Эта версия работает без Render и без Node.js сервера.

## Как устроено

- Сайт публикуется через **GitHub Pages**.
- Каталог товаров хранится в `data/products.json`.
- Тексты и блоки сайта хранятся в `data/site.json`.
- Фото лежат в `uploads/` и `assets/`.
- Админка открывается по адресу `admin.html`.
- При входе в админку вы вручную вводите GitHub token.
- Token не зашит в код сайта и не хранится в файлах. Он используется только в текущей открытой странице браузера.

## Важное ограничение

Это статическая версия. GitHub Pages не запускает backend, поэтому:

- автоматическая отправка Telegram/email с сайта не работает;
- сохранение заказов в `orders.json` автоматически не работает;
- заказ на сайте формируется через email-ссылку: открывается письмо на адрес магазина, а текст заказа копируется в буфер обмена.

Зато работает:

- редактирование товаров;
- добавление товаров;
- удаление товаров;
- загрузка фото;
- редактирование текстов и блоков сайта;
- сохранение изменений прямо в GitHub через GitHub API.

# Пошаговая настройка с нуля

## 1. Создайте новый репозиторий

1. Откройте GitHub.
2. Нажмите **New repository**.
3. Название, например:

```text
lavka-svezhego-myasa
```

4. Можно выбрать **Public** или **Private**.
5. Нажмите **Create repository**.

## 2. Загрузите файлы сайта

Распакуйте архив и загрузите все файлы в репозиторий.

Структура должна быть примерно такая:

```text
index.html
admin.html
admin-github.js
app.js
styles.css
data/
assets/
uploads/
```

Важно: `index.html` должен лежать в корне репозитория.

## 3. Включите GitHub Pages

1. Откройте репозиторий.
2. Перейдите в **Settings**.
3. Слева выберите **Pages**.
4. В блоке **Build and deployment** выберите:
   - Source: **Deploy from a branch**
   - Branch: **main**
   - Folder: **/root**
5. Нажмите **Save**.
6. Подождите 1–3 минуты.
7. GitHub покажет ссылку сайта, например:

```text
https://USERNAME.github.io/lavka-svezhego-myasa/
```

## 4. Создайте GitHub token для админки

1. Откройте GitHub.
2. Нажмите на аватар справа сверху.
3. Перейдите в **Settings**.
4. Слева внизу откройте **Developer settings**.
5. Откройте **Personal access tokens**.
6. Выберите **Fine-grained tokens**.
7. Нажмите **Generate new token**.
8. Задайте имя, например:

```text
Lavka admin token
```

9. Выберите срок действия. Лучше не делать бессрочный, например 30 или 90 дней.
10. В **Repository access** выберите **Only select repositories**.
11. Выберите только репозиторий сайта.
12. В **Repository permissions** поставьте:
    - **Contents: Read and Write**
13. Нажмите **Generate token**.
14. Скопируйте token и сохраните в надёжном месте.

## 5. Вход в админку

Откройте:

```text
https://USERNAME.github.io/lavka-svezhego-myasa/admin.html
```

Заполните:

```text
Owner: USERNAME
Repository: lavka-svezhego-myasa
Branch: main
GitHub token: ваш token
```

Нажмите **Подключиться к GitHub**.

## 6. Как редактировать товары

В админке откройте вкладку **Товары**.

Можно:

- изменить название;
- изменить категорию;
- изменить цену;
- изменить описание;
- загрузить новое фото;
- добавить новый товар;
- удалить товар.

После сохранения админка делает commit в GitHub в файл:

```text
data/products.json
```

Если загружается фото, оно сохраняется в:

```text
uploads/
```

## 7. Как редактировать сайт

Откройте вкладку **Редактирование сайта**.

Можно менять:

- название;
- слоган;
- главный заголовок;
- главный текст;
- телефон;
- email;
- адреса;
- преимущества;
- весь `site.json` через JSON-редактор;
- загружать новые фото.

Для вставки фото в блок сайта:

1. Загрузите фото в блоке **Загрузка фото**.
2. Скопируйте путь вида:

```text
uploads/example-123456.jpg
```

3. Вставьте этот путь в JSON в нужное поле `image`.

## 8. Когда изменения появятся на сайте

После сохранения GitHub Pages обновляет сайт не мгновенно. Обычно нужно подождать от нескольких секунд до пары минут.

Если изменения не видны:

1. Подождите 1–3 минуты.
2. Обновите страницу с Ctrl+F5.
3. Проверьте, появился ли commit в GitHub.
4. Проверьте, что файл `data/products.json` или `data/site.json` действительно изменился.

## 9. Безопасность

Не вставляйте token в код сайта.

Правильный режим:

- token вводится вручную в `admin.html`;
- token не сохраняется в файлах;
- token имеет доступ только к одному репозиторию;
- token имеет только permission `Contents: Read and Write`;
- token периодически перевыпускается.

Если token случайно попал в публичный доступ:

1. GitHub → Settings → Developer settings.
2. Personal access tokens.
3. Удалите старый token.
4. Создайте новый.

## 10. Как работают заказы в этой версии

Так как сервер отсутствует, заказ не отправляется автоматически через Telegram/email API.

При оформлении заказа:

1. Формируется номер заказа.
2. Текст заказа копируется в буфер обмена.
3. Открывается email-письмо на адрес магазина.
4. Покупатель отправляет письмо.

Если позже понадобится автоматическая отправка Telegram/email и запись заказов, нужно вернуть backend: Render, Railway, VPS или другой Node.js сервер.


## В этой версии восстановлена расширенная админка

В разделе **Редактирование сайта** можно редактировать:

- основные тексты;
- главное фото;
- логотип;
- фото доставки;
- верхние маленькие карточки;
- большие карточки раздела;
- дополнительные статьи/блоки;
- услуги доставки;
- весь `site.json`;
- загружать фото в `uploads/`.

## Почему нет автоматического Telegram-уведомления

Эта сборка работает только на GitHub Pages, то есть без backend-сервера. Telegram Bot API требует секретный bot token. Если положить token в браузерный JavaScript, его сможет увидеть любой посетитель сайта. Поэтому автоматическое Telegram-уведомление в GitHub Pages-версии отключено.

Для автоматических Telegram/email-уведомлений нужна серверная версия сайта: Render, VPS, Railway или другой Node.js backend.
