# Мясная лавка — витрина с админкой и GitHub-хранением данных

Проект подготовлен как аналогичная по логике интернет-витрина мясного магазина: главная страница, категории, карточки товаров, корзина, оформление заказа, административный вход `/admin`, загрузка фото и уведомления о заказах.

## Что внутри

- `public/index.html` — витрина магазина.
- `public/admin.html` — административная панель по адресу `/admin`.
- `data/products.json` — товары каталога.
- `data/site.json` — контакты, доставка, тексты и преимущества.
- `data/orders.json` — заказы.
- `public/uploads/` — изображения товаров.
- `server.js` — API для каталога, заказов, админки, загрузки фото, Telegram/email-уведомлений.

## Быстрый локальный запуск

```bash
npm install
cp .env.example .env
npm start
```

Откройте:

- сайт: `http://localhost:3000`
- админка: `http://localhost:3000/admin`

Локальный пароль по умолчанию: `admin123`. Для реального использования обязательно задайте в `.env`:

```env
ADMIN_PASSWORD=ваш-сложный-пароль
SESSION_SECRET=длинная-случайная-строка
```

## Режимы хранения данных

### 1. Local — локальные JSON-файлы

Подходит для теста и разработки.

```env
DATA_DRIVER=local
```

При изменении товара через админку сервер меняет файлы в папке `data/`, а фото сохраняет в `public/uploads/`.

### 2. GitHub — данные и фото сохраняются коммитами в репозиторий

Подходит для рабочей версии, где каталог и изображения должны храниться на GitHub.

```env
DATA_DRIVER=github
GITHUB_OWNER=your-org-or-user
GITHUB_REPO=your-repo
GITHUB_BRANCH=main
GITHUB_TOKEN=github_pat_xxx
```

Создайте fine-grained GitHub token с доступом только к нужному репозиторию и правом `Contents: Read and Write`. Токен нельзя хранить в браузере или фронтенде — только в переменных окружения backend-сервера.

## Почему нельзя сделать безопасную админку только на GitHub Pages

GitHub Pages отдаёт статические файлы и не умеет безопасно хранить `ADMIN_PASSWORD`, `GITHUB_TOKEN`, SMTP-пароль и Telegram-токен. Поэтому публичный сайт может быть статическим, но для `/admin`, загрузки фото, записи заказов и уведомлений нужен backend/API.

Практичные варианты:

1. **Этот Node.js сервер на Render/Railway/VPS/Vercel Serverless** — проще всего. Публичный сайт и админка работают через один домен.
2. **GitHub Pages + отдельный API** — сайт можно оставить на GitHub Pages, а `/api/*` направить на backend. Потребуется настроить CORS и URL API во фронтенде.
3. **Decap CMS + Git Gateway** — хороший вариант для статичного каталога, но корзина, заказы и уведомления всё равно требуют отдельного обработчика.

## Уведомления о заказах

Уведомления уже внедрены в backend. После оформления заказа сервер сохраняет заказ и пытается отправить сообщение:

- на email `izhmeat@gmail.com`;
- в Telegram, если подключён бот.

Подробная пошаговая инструкция вынесена в файл `NOTIFICATIONS-SETUP.md`. Токен Telegram-бота уже добавлен в `.env`; осталось получить и заполнить `TELEGRAM_CHAT_ID`.

### Email

В `.env.example` уже указан получатель:

```env
MAIL_TO=izhmeat@gmail.com
```

Для отправки писем заполните SMTP-настройки:

```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your-gmail@gmail.com
SMTP_PASS=your-gmail-app-password
MAIL_FROM="Лавка свежего мяса <your-gmail@gmail.com>"
MAIL_TO=izhmeat@gmail.com
```

### Telegram

Создайте бота через `@BotFather`, получите токен и chat ID, затем заполните:

```env
TELEGRAM_BOT_TOKEN=123456:ABC...
TELEGRAM_CHAT_ID=123456789
```

В админке появился блок **«Уведомления о заказах»** с кнопкой тестовой отправки.

## Возможности админки `/admin`

- вход по паролю;
- просмотр списка товаров;
- добавление товара;
- редактирование товара;
- загрузка фотографии;
- удаление товара;
- просмотр последних заказов;
- автоматическая запись изменений в локальные файлы или GitHub.

## Подготовка репозитория GitHub

1. Создайте новый репозиторий.
2. Загрузите все файлы проекта.
3. Убедитесь, что в репозитории есть папки `data/` и `public/uploads/`.
4. Создайте `.env` на сервере деплоя, но не коммитьте его в GitHub.
5. Включите режим `DATA_DRIVER=github`.
6. Запустите сервер.

## Запуск через файл

### Windows
Откройте двойным кликом:

```text
START-WINDOWS.bat
```

Файл сам создаст `.env`, установит зависимости при первом запуске и откроет `http://localhost:3000`.

### macOS
Откройте:

```text
START-MAC.command
```

Если macOS заблокирует запуск, нажмите правой кнопкой мыши по файлу → **Открыть**.

### Linux
Запустите:

```bash
./START-LINUX.sh
```

Админка доступна по адресу `http://localhost:3000/admin`, пароль по умолчанию — `admin123`.


## Запуск на Windows двойным кликом

1. Распакуйте архив в обычную папку.
2. Запустите `START-WINDOWS.bat`.
3. Если не сработало, запустите `RUN-WINDOWS.cmd`.

Windows-файл запуска специально сделан без кириллицы и с CRLF-строками, чтобы `cmd.exe` не выдавал ошибки `not recognized as an internal or external command`.


## Формат номера заказа

Номер заказа формируется автоматически в формате:

```text
дд-мм-гггг-номер_по_порядку_за_день
```

Пример:

```text
07-06-2026-17
```

где `17` — семнадцатый заказ, оформленный в этот день.
