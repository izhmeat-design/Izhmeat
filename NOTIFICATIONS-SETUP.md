# Настройка уведомлений о заказах

Сайт уже умеет отправлять уведомления после оформления заказа:

- на email: `izhmeat@gmail.com`;
- в Telegram-бота или Telegram-группу.

Уведомления включаются через файл `.env` на сервере. Не добавляйте `.env` в публичный GitHub-репозиторий.


## Текущий Telegram-токен

Токен бота уже добавлен в файл `.env` в строку `TELEGRAM_BOT_TOKEN`. Полный токен не выводится на сайте и не показывается в админке.

`TELEGRAM_CHAT_ID` уже добавлен в `.env`: `8674762889`.

1. Откройте созданного бота в Telegram.
2. Нажмите **Start**.
3. Отправьте боту любое сообщение, например `тест`.
4. Откройте админку сайта → **Редактирование сайта** → **Уведомления о заказах**.
5. Нажмите **Найти Telegram Chat ID**.
6. Скопируйте найденное число в `.env` в строку:

```env
TELEGRAM_CHAT_ID=сюда-вставить-id
```

7. Перезапустите сервер.
8. Нажмите **Отправить тестовое уведомление**.


## 1. Email-уведомления на izhmeat@gmail.com

В `.env` уже предусмотрен получатель:

```env
MAIL_TO=izhmeat@gmail.com
```

Чтобы сайт мог отправлять письма, нужен SMTP-ящик-отправитель. Проще всего использовать Gmail:

```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=ваш-gmail@gmail.com
SMTP_PASS=пароль-приложения-gmail
MAIL_FROM="Лавка свежего мяса <ваш-gmail@gmail.com>"
MAIL_TO=izhmeat@gmail.com
```

Для Gmail нужен не обычный пароль от почты, а пароль приложения. Обычно порядок такой:

1. Включите двухфакторную защиту в Google-аккаунте.
2. Откройте настройки безопасности Google.
3. Создайте пароль приложения для почты.
4. Скопируйте этот пароль в `SMTP_PASS`.
5. Перезапустите сервер сайта.

Если SMTP не заполнен, заказ всё равно сохранится, но письмо отправлено не будет.

## 2. Создание Telegram-бота через BotFather

1. Откройте Telegram.
2. Найдите официального бота `@BotFather`.
3. Нажмите **Start**.
4. Отправьте команду:

```text
/newbot
```

5. Введите название бота, например:

```text
Лавка свежего мяса — заказы
```

6. Введите username бота. Он должен заканчиваться на `bot`, например:

```text
lavka_svezhego_myasa_orders_bot
```

7. BotFather выдаст токен вида:

```text
123456789:AA...
```

8. Скопируйте его в `.env`:

```env
TELEGRAM_BOT_TOKEN=123456789:AA...
```

## 3. Получение TELEGRAM_CHAT_ID

### Вариант А: уведомления лично вам

1. Откройте созданного бота в Telegram.
2. Нажмите **Start**.
3. Отправьте боту любое сообщение, например:

```text
тест
```

4. Откройте в браузере ссылку, заменив токен на свой:

```text
https://api.telegram.org/botВАШ_ТОКЕН/getUpdates
```

5. Найдите в ответе поле:

```json
"chat":{"id":123456789}
```

6. Скопируйте число в `.env`:

```env
TELEGRAM_CHAT_ID=123456789
```

### Вариант Б: уведомления в группу

1. Создайте Telegram-группу для заказов.
2. Добавьте туда своего бота.
3. Напишите любое сообщение в группу.
4. Откройте:

```text
https://api.telegram.org/botВАШ_ТОКЕН/getUpdates
```

5. Найдите `chat.id`. У групп он часто начинается с минуса, например:

```env
TELEGRAM_CHAT_ID=-1001234567890
```

6. Сохраните `.env` и перезапустите сервер.

## 4. Проверка в админке

1. Запустите сайт.
2. Откройте:

```text
http://localhost:3000/admin
```

3. Войдите в админку.
4. Перейдите в раздел **Редактирование сайта**.
5. Найдите блок **Уведомления о заказах**.
6. Нажмите **Отправить тестовое уведомление**.

Если всё настроено правильно, тест придёт на email и/или в Telegram.

## 5. Что происходит при реальном заказе

После оформления заказа сервер:

1. проверяет корзину;
2. сохраняет заказ в `data/orders.json` или GitHub-хранилище;
3. отправляет уведомление в Telegram, если заполнены `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`;
4. отправляет email на `izhmeat@gmail.com`, если заполнены SMTP-настройки.


## Если Telegram показывает ошибку соединения

Если в админке появляется ошибка соединения с Telegram API, проверьте:

1. Сервер или компьютер, где запущен сайт, должен иметь доступ к `api.telegram.org`.
2. Firewall, антивирус, корпоративная сеть или провайдер не должны блокировать Telegram API.
3. Если сайт запущен локально и Telegram API недоступен, попробуйте включить VPN или запустить сайт на сервере, где Telegram API доступен.
4. Если ошибка `chat not found`, заново нажмите **Start** в боте, отправьте ему сообщение и проверьте `TELEGRAM_CHAT_ID`.


## Если сервер не подключается к Telegram API

Если в админке ошибка вида `Таймаут соединения с api.telegram.org`, значит сервер не может достучаться до Telegram API. В этом случае есть два рабочих варианта:

### Вариант 1. Запустить сайт на сервере, где Telegram API доступен

Например, на VPS/хостинге, где `api.telegram.org` не блокируется.

### Вариант 2. Подключить HTTPS-прокси

В `.env` добавьте:

```env
TELEGRAM_PROXY_URL=http://user:password@host:port
```

или, если прокси без логина и пароля:

```env
TELEGRAM_PROXY_URL=http://host:port
```

После изменения `.env` перезапустите сервер и снова нажмите в админке **Отправить тестовое уведомление**.

Важно: обычные SOCKS5-прокси этой сборкой не используются. Нужен HTTP/HTTPS-прокси.


## Статус после оформления заказа

После сохранения заказа покупатель сразу видит сообщение:

```text
Заказ отправлен и принят в обработку. Спасибо за заказ.
```

Telegram/email-уведомления отправляются в фоне и больше не держат корзину в состоянии `Отправляем заказ...`.
